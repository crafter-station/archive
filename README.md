# WhatsApp Community Archive

A Next.js app that archives a WhatsApp community chat, stores media, and runs a scheduled AI agent that turns chat activity into daily logs, memories, ships, resources, and events.

The public UI is centered on the daily archive:

- `/` redirects to today's community day view.
- `/YYYY-MM-DD` shows logs, ships, resources, and events for that day.

## Stack

- Next.js 16 App Router
- React 19
- Drizzle ORM
- Neon Postgres
- Trigger.dev scheduled tasks
- AI SDK with OpenAI
- Vercel Blob for media storage
- Tailwind CSS v4
- Bun

## Setup

Install dependencies:

```bash
bun install
```

Create a local env file:

```bash
cp .env.example .env
```

Fill in the required values:

```bash
DATABASE_URL="postgresql://..."
OPENAI_API_KEY="sk-proj_..."
BLOB_READ_WRITE_TOKEN="vercel_blob_rw_..."
WHATSAPP_WEBHOOK_SECRET="..."
```

Apply database migrations:

```bash
bun run db:migrate
```

Start the app:

```bash
bun run dev
```

Open `http://localhost:3000`.

## WhatsApp Webhook

Webhook endpoint:

```txt
POST /api/webhooks/whatsapp
```

The route validates the `x-webhook-signature` header against `WHATSAPP_WEBHOOK_SECRET` and persists incoming group messages, replies, reactions, and media.

Stored chat data lives in these tables:

- `participants`
- `messages`
- `message_media`
- `message_reactions`
- `webhook_events`

The webhook currently filters to the configured group JID in `src/lib/whatsapp-constants.ts`.

## Scheduled Log Agent

The Trigger.dev task is defined in:

```txt
src/trigger/chat-log-agent.ts
```

It runs every 30 minutes in `America/Bogota` time.

For each run, it:

- Computes the current 30-minute log window.
- Reads messages from the beginning of the previous window through the end of the current window.
- Injects active memories and existing structured entities into the prompt using TOON.
- Lets the LLM call tools to create or update memories, ships, resources, and events.
- Stores a short user-facing summary in `logs.final_response`.

Run Trigger.dev locally:

```bash
bunx trigger.dev@latest dev
```

Deploy Trigger.dev tasks with the Trigger.dev CLI or dashboard flow used by the project.

## Day View

Daily archive pages live at:

```txt
/2026-05-09?tab=logs
/2026-05-09?tab=ships
/2026-05-09?tab=resources
/2026-05-09?tab=events
```

Date grouping uses `America/Bogota`:

- Logs, ships, and resources are grouped by the day they were detected or shared.
- Events are grouped by the day they occur.

The page includes a weekly calendar strip for moving between dates.

## Data Model

Core AI-generated archive tables:

- `logs`: 30-minute public summaries generated by the scheduled agent.
- `memories`: durable facts about the community used as agent context.
- `ships`: live internet artifacts built and shared by community members.
- `resources`: useful links, tutorials, documentation, books, or products.
- `events`: dated community events with UTC timestamps and local timezone metadata.

All actual timestamps are stored as timezone-aware UTC values. Event records also keep interpreted local date/time fields for display and ambiguity handling.

## Development Notes

- Do not commit `.env`; it is intentionally ignored.
- Run `bun run db:generate` after schema changes.
- Run `bun run lint` and `bun run build` before shipping changes.
- The scheduled agent requires `OPENAI_API_KEY` both locally and in the Trigger.dev runtime environment.
- The app requires `DATABASE_URL` at build/runtime because server routes and pages validate environment variables.
